My projects

bizstore.si

  • Implementing technologies:

    Web application runs on AWS EC2 instance and is deployed with ["fabric"](http://docs.fabfile.org/en/1.4.3/) library into development, integration and production environment. It is built within Django 1.3 framework. Data migrations are managed with third party application ["south"](http://south.readthedocs.org/en/0.7.6/), which is performing schema changes on DB. Python three engine [mptt](https://github.com/django-mptt/django-mptt), was used to provide hierarchy to the services pages. For improving performance, we used [django_compressor](https://github.com/jezdez/django_compressor)

  • Overview:

    It's built according to the requirements of a telecom company, which had recently entered cloud brokerage market. The goal is to provide SOHO and SMEs with commodity like cloud services. Application offers a cloud based ordering and managing of those services, thus providing easy access with online product like shopping cart. At the back end is made as a custom CMS, to provide easy managing of the front end. Templates are selected by the system according to the url that was requested. If template with the url's name exists than it is displayed otherwise generic template is displayed. With this feature, we enabled designers to work with easy to use templating system.

  • Features:

    1. Social login as well as custom login forms. User profile is connected to Django's user.auth module and which is connected to the cloud services gateway. Social login is implemented with the third party component (janrain).
    2. Cloud based services are listed in special categories, for easy access
    3. Price list is automatically generated from range of available services, that are stored in gateway DB. Services also include features that come in bundle with the service and the kind that are orderable.
    4. Shopping cart includes order procedure which contains four steps. For better user experience we included jQuery into process for the purpose of calculating prices on the spot.
    5. Revision of ordered services by clients was implemented in order to provide them with easy to use managing of number of subscriptions for particular service.
    7. Customer is also provided with messaging system, to enable communication between technical stuff and themselves from application directly, but at the same time connecting to telecoms CRM.

  • My role:

    I was involved in the project from the start, as a developer and sometimes front end expert (HTML5, CSS3, Javascript). From implementing logic which presented shopping cart and processes that supported messaging system and many others, my task was also to keep the design responsive, which was also important since we provide access from anywhere, with any device.

URL: bizstore.si

POPP - custom CRM

  • Implementing technologies:

    Application also runs on AWS EC2 instance, written in python with Django 1.3, framework. Connector for the AWS is using boto library.

  • Overview:

    This is an internal application of which intention is to implement custom CRM which is used by the sales staff of a telecom company, which is offering cloud based subscriptions to services. Within application sales staff is enabled to take the client through a process of ordering to a subscription of a cloud based service. At the end a salesperson can print all the legal documents and quote in order for a client to sign it. Application also generates reports for management to have access to information of sales personnel's success, which is implemented with google charts javascript library. Within the application a salesperson is also able to lookup domain availability, since one of the offers is domain registration. Sales staff is also able to send quote directly to clients email address.

  • Features:

    Sales is able to store clients data in db, which is then linked to their account in order to easily calculate commissions. Quote editor is enabling sales to make taylormade quotes for their clients, ordering services. Quotes that had been made are easily sent to the client directly from within application via email, where every email is seemingly sent from the salesperson address. We achieved this by using AWS SES and a boto library to connect addresses of the staff to AWS Pricelist is providing them with easy access to the information which is usually needed quick Media files are served from within the application, in order for sales to show or print them out and live them at the potential client to contemplate From application management is able to see the success of the sales personnel on how much quotes have been made, sent to the clients and how many quotes had been realized. We also built in a dynamic tool written with javascript using MVVM framework Knockout.js, which helps staff to calculate expenses of a compatible offers in order to show clients how much they can gain with the subscription with them Domain availability check tool was incorporated in to application to enable staff to check for available domain names which client wishes on the spot. User don’t have to be computer savy in order to sell services. Messaging system, which is implemented in order for the staff to talk to technical staff about a particular client and make notes about them. Generating documents in PDF format enabled staff to put out quick quote, which could client turn into a contract on the spot if they wished

  • My role:

    I was involved in this project from the beginning to the end (altho is still a work in progress, as any software that is alive). My responsibility was to implement features like dynamic UI for the domain availability, building price comparison tool, implementing connection to AWS SES, generation pricelist directly from db and implementing the messaging system.

unfortunately this is an internal company application and not available to the general public, so I can not provide url.

Sales app

  • Technologies:

Unfortunately this is an internal company application and as such not available to the general public, so I can not provide url.
